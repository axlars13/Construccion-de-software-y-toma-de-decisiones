Preguntas

1. ¿Por qué es buena práctica validar con JavaScript?

Porque permite verificar los datos antes de enviarlos al servidor y evita errores simples.

2. ¿Cómo se pueden saltar esas validaciones?

Se pueden saltar desactivando JavaScript o modificando el código desde el navegador.

3. Si se pueden saltar, ¿por qué sigue siendo buena práctica?

Porque mejora la experiencia del usuario y ayuda a detectar errores rápidamente.
